为什么要优化vps服务器内存

程序员都喜欢折腾vps服务器，但是又不想花很多钱在上面，所以总是买一些非常低配的vps，512M内存的服务器又要跑mysql，又要mongodb，还有docker,wordpress

想玩一玩的都装上，这样就根本不够用了, 而且总是有程序进程被莫名其妙kill掉，总是查看到如下日志。

1
2

Aug 23 21:51:30 DBSlave-live kernel: [43560651.172427] Out of memory: Kill process 16255 (mysqld) score 946 or sacrifice child
Aug 23 21:51:30 DBSlave-live kernel: [43560651.216883] Killed process 16255, UID 500, (mysqld) total-vm:6367844kB, anon-rss:3631300kB, file-rss:3196kB

上面的错误日志是因为Linux有一种机制叫OOM killer（Out Of Memory killer），它就是在服务器内存占用高的时候，选择性kill掉一些进程。

下面我就分享几种常用的解决内存问题方法。

防止进程被oom killer 杀掉

首先解决进程经常被干掉的问题。

1
2
3
4
5

echo -10 > /proc/进程pid/oom_score_adj   // 范围是【-17到15】越小月不容易kill，

echo "vm.oom_kill_allocating_task = 1" >> /etc/sysctl.conf   // 意思是内存不足时，不通过扫描来寻找需要kill的进程，节约资源

sysctl  -p

添加swap分区

查看系统中已经配置的swap

使用命令swapon -s即可查看系统中在用的swap

1
2
3

dev@dev:~$ swapon -s
Filename                Type        Size    Used    Priority
/dev/dm-1               partition   524284  0       -1

如果配置有多个swap分区或者文件的话，这里将会有多行，每行代表一个正在被系统使用的swap分区或文件，下面是每个字段的意思：

  • Filename：如果swap类型是分区，这里将是分区的路径，如果swap类型是文件，这里将是文件的路径
  • Type：swap的类型，partition代表这是一个swap分区，file代表这是一个swap文件
  • Size：swap的大小，单位是k，这里524284表示的差不多是512M
  • Used：已经被使用的大小，这里0表示还没有被使用到
  • Priority：优先级，优先级高的swap将会被优先使用，同等优先级的swap将会被均匀的使用（round-robin算法），优先级可以通过“swapon -p”命令来设置
